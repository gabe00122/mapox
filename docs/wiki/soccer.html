Soccer

Two teams play on an open gridworld field with goals at opposite ends. Push the ball into the opponent's goal to score. All agents act simultaneously with randomized execution order.

Game Rules

Field

  • Open rectangle with no internal walls or obstacles.
  • Goals are 1-tile-deep zones spanning goal_width tiles, centered on opposite edges.
  • Bottom goal (defended by Red) and top goal (defended by Blue).

Teams

Agents are split evenly: first half Red, second half Blue.

  • Red team: starts in the bottom quarter, centered horizontally.
  • Blue team: starts in the top quarter, centered horizontally.

The Ball

  • Single tile rendered as AGENT_BALL (tile ID 16). Starts at field center.
  • When a moving agent's destination is the ball's tile, the ball is pushed 1 tile in that agent's movement direction.
  • If the pushed ball would land on a wall or another agent, the ball stays put and the pushing agent also stays in place (cannot push through a blocked ball).
  • Staying agents (action = STAY) never interact with the ball.

Actions Used

IDAction
0Move up
1Move right
2Move down
3Move left
4Stay

Primary action and dig are masked off.

Turn Execution

  • All agents submit actions simultaneously.
  • A random permutation determines execution order each step.
  • Agents are processed sequentially (via jax.lax.fori_loop) — only the first agent (in random order) to reach the ball pushes it.
  • Agent-to-agent collision: an agent cannot move to a tile occupied by another agent (unless the ball is there, in which case the push rules apply).

Scoring

  • Ball entering the top goal = Red scores (+goal_reward for Red, -goal_reward for Blue).
  • Ball entering the bottom goal = Blue scores (+goal_reward for Blue, -goal_reward for Red).
  • After a goal, ball and all agents reset to starting positions.

Termination

Episode ends after length steps.
