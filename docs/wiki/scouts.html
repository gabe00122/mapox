Scouts

Asymmetric cooperative environment with two roles: Harvesters unlock treasure tiles, Scouts collect unlocked treasures. Requires coordination between different agent types.

Game Rules

Map Generation

The map uses Perlin noise at multiple resolutions with random amplitude weights to create permanent wall patterns. Walls are impassable. Treasure tiles (num_treasures) are scattered on random empty positions.

Agent Roles

Harvesters

  • Move at reduced speed — they can only act once every harvesters_move_every steps (default 6). Between moves they are frozen.
  • When a Harvester walks onto a locked treasure (TILE_FLAG), the treasure becomes unlocked (TILE_FLAG_UNLOCKED).
  • Harvesters receive harvester_reward for touching a locked treasure.
  • Harvesters start at the same position as Scouts.

Scouts

  • Move at full speed every step.
  • When a Scout walks onto an unlocked treasure (TILE_FLAG_UNLOCKED), the treasure is removed from the map.
  • Scouts receive scout_reward for collecting an unlocked treasure.
  • Scouts cannot interact with locked treasures.
Coordination: Harvesters must unlock treasures before Scouts can collect them. Since Harvesters move slowly, efficient play requires Scouts to plan routes around already-unlocked treasures while Harvesters push deeper into the map.

Actions Used

IDAction
0Move up
1Move right
2Move down
3Move left

Only 4 movement directions. Stay, primary action, and dig are masked off.

Turn Order

Harvesters process first (unlocking treasures), then Scouts process (collecting unlocked treasures). This means a Harvester and Scout on the same treasure in the same step will correctly unlock then collect.

Rewards

  • Harvester: +harvester_reward (default 1.0) when stepping onto a locked treasure.
  • Scout: +scout_reward (default 1.0) when stepping onto an unlocked treasure.

Termination

Episode ends after length steps.
