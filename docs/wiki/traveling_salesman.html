Traveling Salesman

Multiple flags are scattered on an open map. Each agent independently tracks which flags it has visited. After collecting all flags, the agent's flag set resets for another round.

Game Rules

Map

An open map (no walls other than the border) with decorative tiles. No destructible walls. Flags are placed on random positions at reset and remain fixed for the episode.

Flag Collection

  • Each agent maintains a personal boolean array tracking which of the num_flags flags it has already visited.
  • Stepping onto a flag tile that the agent has not yet visited grants flag_reward and marks that flag as collected for that agent.
  • Stepping onto an already-visited flag does nothing.
  • When an agent has collected all flags, its tracking array resets — all flags become available again for a new round.
Route optimization: The challenge is to find an efficient visitation order. With multiple agents, agents may develop strategies to cover different regions — though each agent tracks flags independently.

Actions Used

IDAction
0Move up
1Move right
2Move down
3Move left
4Stay

5 actions available. Primary action and dig are masked off.

Movement

  • Moving into a wall (border) blocks the agent — it stays in place.
  • No collision between agents — multiple agents can occupy the same tile.

Rewards

  • +flag_reward (default 1/3) per new flag visited.
  • No penalty for revisiting flags or staying.

Termination

Episode ends after length steps.

Observations

Standard 4-channel format. Only tile channel is used.

Partial observability note: Agents can see flag tiles in their local view, but cannot distinguish between flags they have already collected and those they haven't. This makes route planning challenging under partial observation.
